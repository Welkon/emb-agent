# Task Execution Contract

- When an active task exists, re-read its `implement` context before editing and add missing context files if the write surface expanded.
- Keep the task status accurate. Activate the task before major work and resolve it only after evidence or follow-up notes exist.
- Preserve context hygiene. If the working set becomes large, pause or resume through emb-agent instead of relying on raw chat continuity.
- When a task changes hardware truth, requirements truth, or verification evidence, update those artifacts in the same workstream.
- Prefer self-contained closure notes: what changed, what was verified, what remains unknown.
