## Steps

1. Confirm the rule gap with a real task outcome.
2. Search first and confirm whether the rule already exists.
3. Classify the problem as missing, outdated, retired, or ignored.
4. If the problem is an outdated rule, update it immediately.
5. If the problem is a missing rule, apply the 2/3 recording threshold before adding it.
6. If the problem is a retired rule, remove it, scope it, or mark it deprecated.
7. If the problem is an ignored rule, improve where it surfaces on the normal task path.
8. Record the reason the previous rule set was insufficient.
9. Re-run the affected verification command.
10. If the lesson is high-cost, also activate it in the workflow or shell path that should naturally surface it next time.
